# StorageSign 実動作確認チェックリスト

このチェックリストは、搬送ブロック対応（ホッパー/ドロッパー/ディスペンサー/クラフター/ホッパー付きトロッコ）追加後の実サーバー検証向けです。

## 0. 事前準備

- [ ] サーバーを停止し、ビルド済み JAR を `plugins` に配置する
- [ ] `config.yml` の `auto-import: true` / `auto-export: true` を確認する
- [ ] 検証用ワールドで以下を準備する
  - [ ] 隣接コンテナ（チェスト等）
  - [ ] 対応アイテム登録済み StorageSign
  - [ ] ホッパー、ドロッパー、ディスペンサー、クラフター、ホッパー付きトロッコ

## 1. 基本回帰（既存ホッパー）

- [ ] ホッパーでコンテナへ投入したとき、満杯分が SS に吸収される
- [ ] ホッパーでコンテナから搬出したとき、1 ティック後に SS から補充される
- [ ] ログに異常（例外、NPE）が出ない

## 2. Inventory 間搬送（新規対象）

### 2.1 ドロッパー

- [ ] SS 隣接コンテナへ搬送したとき、インポート（吸収）が機能する
- [ ] SS 隣接コンテナから結果的に搬出される経路で、エクスポート（補充）が機能する

### 2.2 ディスペンサー

- [ ] SS 隣接コンテナへの搬送でインポートが機能する
- [ ] SS 隣接コンテナからの搬出経路でエクスポートが機能する

### 2.3 クラフター

- [ ] クラフター排出サイクルで SS 在庫同期が機能する
- [ ] 連続駆動でも在庫表示と実在庫が乖離しない

### 2.4 ホッパー付きトロッコ

- [ ] SS 近傍通過時に搬送経路として認識される
- [ ] インポート/エクスポート動作が固定ホッパー経路と同様に機能する

## 3. ワールド吐き出し（BlockDispenseEvent）

- [ ] ドロッパーのワールド吐き出し時、SS 在庫が減算される
- [ ] ディスペンサーのワールド吐き出し時、SS 在庫が減算される
- [ ] クラフターのワールド吐き出し時、SS 在庫が減算される
- [ ] SS 残量不足時でもイベントが停止せず、同期可能分のみ減算される

## 4. 非対応ソースの確認

- [ ] かまど等の非対応ブロックでは SS 同期処理が走らない
- [ ] 既存の手動入出庫、autocollect、クラフト機能に回帰がない

## 5. 境界ケース

- [ ] ラージチェスト（DoubleChest）でも隣接判定が崩れない
- [ ] 看板の向き（立て看板/壁看板/吊り看板）で判定漏れがない
- [ ] サーバー再起動後に SS 表示値が正しい

## 6. 完了判定

- [ ] 上記チェック項目で blocker なし
- [ ] 既知の警告ログのみ（新規例外なし）
- [ ] 運用ワールド投入可と判断できる
